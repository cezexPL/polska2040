# Bezpieczeństwo informacji

## Dozwolony zakres

Repozytorium przechowuje wyłącznie jawne źródła, jawne analizy i syntetyczne szablony. Prywatne repozytorium GitHub nie jest systemem przeznaczonym do informacji niejawnych.

## Zakazane treści

- informacje niejawne lub pochodzące z nieautoryzowanego ujawnienia;
- dokładne podatności jednostek, infrastruktury lub systemów;
- częstotliwości, konfiguracje, parametry użycia i procedury operacyjne;
- szczegółowe instrukcje budowy lub użycia uzbrojenia;
- lokalizacje wrażliwych zasobów niewynikające z potrzeb jawnej analizy;
- dane osobowe niewymagane do oceny publicznej funkcji;
- loginy, hasła, tokeny, klucze i adresy zawierające poświadczenia.

## Aneks ograniczony

Repozytorium zawiera tylko jego strukturę, pytania, wymagane klasy danych i zasady dostępu. Rzeczywisty aneks może powstać wyłącznie w autoryzowanym środowisku i poza historią Git.

## Sekrety

Sekretów nie przekazuje się w czacie, commitach, issue ani dokumentach. Gdy przekazanie będzie konieczne, należy użyć jednorazowego linku z `https://secret.notangrysoftware.com`.

## Zgłaszanie problemu

Nie otwieraj publicznego issue. Zatrzymaj publikację, usuń materiał z bieżącej gałęzi w sposób zachowujący dowody audytowe poza repo i powiadom właściciela projektu bez cytowania treści wrażliwej.
